| 类型           | 参数                      | 典型值                                                       | 影响对象                                                     | 改大后的影响                                   |
| -------------- | ------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | ---------------------------------------------- |
| 词表与序列长度 | `vocab_size`              | 6400                                                         | 词嵌入层、输出层                                             | 增加词表覆盖范围，但增大模型体积和内存占用     |
|                | `max_position_embeddings` | 32768                                                        | 位置编码、注意力机制                                         | 支持更长上下文，但增加计算复杂度和显存消耗     |
| 模型宽度和深度 | `hidden_size`             | 768                                                          | 所有Transformer层的隐藏维度                                  | 增强模型表达能力，但显著增加参数量和计算量     |
|                | `num_hidden_layers`       | 8                                                            | Transformer层数                                              | 提升模型深度，增强特征提取能力，但训练更不稳定 |
|                | `intermediate_size`       | `math.ceil(hidden_size * math.pi / 64) * 64`.                | FFN层隐藏维度,64是为了对齐GPU的内存访问，Pi是数学美感+经验公式？ | 增强非线性变换能力，增加单层计算量             |
|                | `rms_norm_eps`            | 1e-6                                                         | RMS归一化稳定性                                              | 过小可能导致数值不稳定                         |
| Attention 相关 | `num_attention_heads`     | 8                                                            | Multi-Head Attention头数                                     | 增强并行注意力能力，提升建模能力               |
|                | `num_key_value_heads`     | 4                                                            | KV头数（MQA/GQA）                                            | 减少KV缓存，提升推理效率                       |
|                | head_dim                  | 96 (768/8)                                                   | 单头注意力维度                                               | 增大可捕捉更细粒度特征，需配合hidden_size调整  |

|                | rope_scaling              | {"beta_fast": 32, "beta_slow": 1, "factor": 16, "original_max_position_embeddings": 2048, "attention_factor": 1.0, "type": "yarn"} | RoPE缩放精度                                                 | 需配合inference_rope_scaling使用               |
| MoE 相关       | use_moe                   |                                                              |                                                              |                                                |
|                | num_experts               | 4                                                            | 专家数量                                                     | 增加专家多样性，提升模型容量                   |

### MiniMindForCausalLM

下午梳理模型类关系

```
MiniMindForCausalLM
→ MiniMindModel
→ MiniMindBlock
→ Attention / FeedForward 只看接口
```



MiniMindForCausalLM 是最外层的model，包括主干模型和LLM的输入输出头

```
1. 调用 MiniMindModel 得到 hidden_states
2. 使用 lm_head 把 hidden_states 映射到 vocab_size
3. 输出 logits
4. 如果训练时传入 labels，可能还会计算 loss
```

MiniMindForCausalLM 的作用：

它不是一个新的 Transformer，而是在 MiniMindModel 外面包了一层 Causal LM 任务头。负责前向传播，循环前向传播，筛选每次循环的token，生成序列

### MiniMindModel

照旧看init

然后思考

```
1. embedding 层在哪里？ 
2. layers 是不是 ModuleList？
3. layers 里面放了多少个 block？ # 在MiniMindConfig中的num_hidden_layers，默认为8个block
4. final norm 在哪里？ # hidden_states = self.norm.forward(hidden_states) #过一次循环norm一次
5. forward 返回的是什么？ #隐藏状态、kv缓存，辅助损失
```

流程如下：

```
MiniMindModel.forward()

input_ids: [B, T]
↓
embed_tokens(input_ids): [B, T, H]
↓
presents = []
for layer in layers:
    hidden_states,presents = layer(hidden_states, ...)
    presents.append(present)
↓
norm(hidden_states): [B, T, H]
↓
return hidden_states, presents, ...
```



### MiniMindBlock

重点看它是否是典型 Decoder-only Block：

```
RMSNorm
Self-Attention
Residual
RMSNorm
FeedForward / MoE
Residual

x
│
├── input_layernorm
│
├── self_attn
│
├── residual add
│
├── post_attention_layernorm
│
└── residual add
```

今天不用深入 attention 内部，只要知道：

```
Attention 输入 hidden_states，输出同样维度的 hidden_states
MLP 输入 hidden_states，输出同样维度的 hidden_states
Block 整体输入输出 shape 不变
```

即：

```
Block 输入:  [B, T, H]
Block 输出:  [B, T, H]
```

## 5. forward 主线
input_ids → embedding → blocks → norm → lm_head → logits

```
MiniMindForCausalLM.forward
│
├── MiniMindModel.forward
│   │
│   ├── input_ids
│   │   shape: [B, T]
│   │
│   ├── embed_tokens
│   │   output: [B, T, H]
│   │
│   ├── Transformer layers × N
│   │   │
│   │   ├── RMSNorm
│   │   ├── Attention
│   │   ├── Residual
│   │   ├── RMSNorm
│   │   ├── FeedForward / MoE
│   │   └── Residual
│   │
│   └── final norm
│       output: [B, T, H]
│
└── lm_head
    output logits: [B, T, V]
```

其中

```
B = batch size
T = sequence length
H = hidden_size
V = vocab_size
N = num_hidden_layers
```



## 6. 张量 shape
input_ids: [B, T]
embedding output:[B, T, H]
block output:[B, T, H]
final hidden_states:[B, T, H]
logits:[B, T, V]
