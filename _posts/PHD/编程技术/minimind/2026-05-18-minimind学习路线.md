### MiniMindForCausalLM

下午梳理模型类关系

```
MiniMindForCausalLM
→ MiniMindModel
→ MiniMindBlock
→ Attention / FeedForward 只看接口
```



MiniMindForCausalLM 是最外层的model，包括主干模型和LLM的输入输出头

```
1. 调用 MiniMindModel 得到 hidden_states
2. 使用 lm_head 把 hidden_states 映射到 vocab_size
3. 输出 logits
4. 如果训练时传入 labels，可能还会计算 loss
```

MiniMindForCausalLM 的作用：

它不是一个新的 Transformer，而是在 MiniMindModel 外面包了一层 Causal LM 任务头。负责前向传播，循环前向传播，筛选每次循环的token，生成序列

| ------------------ | -------------- | ---------- | ---------- | ------------------- |

## 4. Embedding 理解

Embedding 矩阵 shape：
[V, H]

参数量：
V × H = vocab_size*hidden_size

作用：
token id 查表得到 token 向量。

## 5. LM Head 理解

LM Head 权重 shape：
[V, H] 或 [H, V]，以 PyTorch Linear 参数存储为准。

输入：
[B,T,H]

输出：
[B,T,V]

作用：
把 hidden state 映射到词表 logits。



## 6. 权重绑定

tie_word_embeddings:
True / False

embed_tokens.weight 和 lm_head.weight 是否共享：
True / False

节省参数量：
V × H

**为什么小模型特别在意vocab size**

因为小模型的embedding 参数占比很高

如

```
如果 vocab_size = 6400
hidden_size = 768

Embedding 参数量：
6400 × 768 = 4,915,200
约 4.9M 参数
```

如果词表扩大到（6.4k->32k）

```
vocab_size = 32000
hidden_size = 768
```

则

```
32000 × 768 = 24,576,000
约 24.6M 参数
```

也就是说，仅词表扩大，Embedding 就多出约 19.7M 参数。对于一个 64M 小模型，这非常明显。

```
对于 MiniMind 这类小模型，vocab_size 不是随便越大越好。
vocab_size 越大，embedding 和 lm_head 参数越多；
如果模型总参数很小，词表会吃掉大量参数预算。
```

重点：register_forward_hook

`register_forward_hook` 是 PyTorch 框架中一个非常核心且实用的工具。简单来说，它是**挂载在神经网络层上的一个“监听器”或“回调函数”**，允许你在不修改模型原始代码的情况下，获取该层在前向传播时的**输出**和**输入**。

```python
import torch
from model.model_minimind import MiniMindConfig, MiniMindForCausalLM

config = MiniMindConfig()
model = MiniMindForCausalLM(config)
model.eval()

def print_shape(name):
    def hook(module, inputs, output):
        if isinstance(output, torch.Tensor):#输出是张量
            print(f"{name}: {tuple(output.shape)}")
        elif isinstance(output, tuple):#输出是元组
            print(f"{name}: tuple")
            for i, x in enumerate(output):
                if isinstance(x, torch.Tensor):
                    print(f"  output[{i}]: {tuple(x.shape)}")
        else:
            print(f"{name}: {type(output)}")#否则打印名字和类型
    return hook

model.model.embed_tokens.register_forward_hook(print_shape("embed_tokens"))#给embed加入钩子函数，钩子钩的是前面的model.model.embed_tokens.
#def print_shape(name):的name只是print一下
model.lm_head.register_forward_hook(print_shape("lm_head"))给lm_head加入钩子函数，打印lm_head

class RMSNorm(torch.nn.Module):#没什么好树洞额，对应公式敲出来
    def __init__(self, dim: int, eps: float = 1e-5):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(dim))

    def norm(self, x):
        return x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)

    def forward(self, x):
        return (self.weight * self.norm(x.float())).type_as(x)
```

## 3. RMSNorm 公式

RMSNorm 的核心是：

```
对每个 token 的 hidden 向量，在 hidden_size 维度上计算均方根，
然后把向量缩放到稳定范围，再乘以可学习参数 weight。
```

公式如下

$RMS(x) = sqrt(mean(x_i^2) + eps)$

$RMSNorm(x) = x / RMS(x) * weight$

注意它和 LayerNorm 不一样：

```
LayerNorm:
先减均值，再除标准差

RMSNorm:
不减均值，只除 RMSs

RMSNorm 只做缩放，不做中心化。
```

RMSNorm 原论文的核心假设就是：LayerNorm 中的 re-centering invariance，也就是减均值带来的不变性，并非总是必要；只保留 re-scaling invariance 也能稳定训练，并减少计算开销。
 PyTorch 官方文档也将 `RMSNorm` 描述为对 mini-batch 输入应用 Root Mean Square Layer Normalization。

## 4. RMSNorm 参数量

RMSNorm 只有一个可学习参数：

```
self.weight = nn.Parameter(torch.ones(dim))
```

hidden_size =768

RMSNorm 个数 = layers+final norm
RMSNorm 总参数量 =(8+1)*768 = 12,288 + 768 = 13,056

```
RMSNorm 参数量很小，不是模型参数量主要来源。

```



## 5. Pre-Norm Block 结构

```
h = x + Attention(RMSNorm(x))
y = h + MLP(RMSNorm(h))

Attention 前先 Norm
MLP 前也先 Norm

def block_forward(x):
    h = x + self_attn(input_layernorm(x))
    y = h + mlp(post_attention_layernorm(h))
    return y
```

整体结构图如：

```
输入 x
│
├── norm_x = RMSNorm(x)
│
├── attn_out = Attention(norm_x)
│
├── h = x + attn_out
│
├── norm_h = RMSNorm(h)
│
├── mlp_out = MLP(norm_h)
│
└── y = h + mlp_out

Norm 不是放在 residual add 后面，
而是放在 Attention / MLP 前面。

我这样理解的，norm后神经网络训练就没毛刺，更平滑
```

**Pre-Norm 和 Post-Norm 对比**

```
Post-Norm:
x → Attention → Add → Norm

Pre-Norm:
x → Norm → Attention → Add
```

对比表：

| 结构      | 顺序              | 常见特点                    |
| --------- | ----------------- | --------------------------- |
| Post-Norm | 子层计算后再 Norm | 早期 Transformer 常见       |
| Pre-Norm  | 子层计算前先 Norm | 现代 LLM 更常见，训练更稳定 |

可以这样理解：

```
Pre-Norm 让 residual 主干路径更“干净”，
梯度可以更容易沿着 x → x + f(norm(x)) 这条路径传播。
```

Day 3不需要证明它，只需要能解释为什么现代 LLM 更偏好 Pre-Norm：

```
1. 深层模型更稳定
2. 梯度传播路径更直接
3. 更适合大规模自回归语言模型训练
```

## 7. RMSNorm vs LayerNorm

LayerNorm:
减均值 + 除标准差

RMSNorm:
不减均值，只除 RMS

公式对比

$RMS(x) = sqrt(mean(x_i^2) + eps)$

$RMSNorm(x) = x / RMS(x) * weight$

vs

$\mu = mean(x_i)$
$\sigma = sqrt(mean((x_i - \mu)^2) + eps)$
$LayerNorm(x) = (x - \mu) / \sigma * weight + bias$

def repeat_kv(x: torch.Tensor, n_rep: int) -> torch.Tensor: #repeat一下，没什么复杂的
    bs, slen, num_key_value_heads, head_dim = x.shape
    if n_rep == 1: return x
    return (x[:, :, :, None, :].expand(bs, slen, num_key_value_heads, n_rep, head_dim).reshape(bs, slen,
                                                                                               num_key_value_heads * n_rep,
                                                                                               head_dim))

class Attention(nn.Module):
    def __init__(self, config: MiniMindConfig):
        super().__init__()
        # 其中 num_key_value_heads 如果为空会等于 num_attention_heads，否则就形成 GQA 结构
        # 如果配置中没有指定（None），则kv默认与 Q 的头数量（num_attention_heads）相同，当它小于 Q 的头数时，就是 GQA；当它等于 1 时，就是 MQA。
        self.num_key_value_heads = config.num_attention_heads if config.num_key_value_heads is None else config.num_key_value_heads
        self.n_local_heads = config.num_attention_heads  # Q的头的数量
        self.n_local_kv_heads = self.num_key_value_heads  # KV 的头数量
        self.n_rep = self.n_local_heads // self.n_local_kv_heads  # 因为 Q 的头数通常多于 KV 的头数，所以在计算 Attention 时，需要将 KV 的特征在头维度上复制，以和 Q 的头数对齐。
        self.head_dim = config.head_dim  # 每个注意力头的维度大小 #总隐藏维度 = num_attention_heads * head_dim。
        self.is_causal = True #设为 True 表示在解码时，当前 Token 只能看到前面的 Token，不能看到未来的 Token
        self.q_proj = nn.Linear(config.hidden_size, config.num_attention_heads * self.head_dim, bias=False)  # 线性映射层得到q
        self.k_proj = nn.Linear(config.hidden_size, self.num_key_value_heads * self.head_dim, bias=False)
        self.v_proj = nn.Linear(config.hidden_size, self.num_key_value_heads * self.head_dim, bias=False)
        self.o_proj = nn.Linear(config.num_attention_heads * self.head_dim, config.hidden_size, bias=False) #前面的都是单个head里面的qkv，这个就是把多个head拼接起来并返回hidden_size
        self.q_norm = RMSNorm(self.head_dim, eps=config.rms_norm_eps)
        self.k_norm = RMSNorm(self.head_dim, eps=config.rms_norm_eps)
        self.attn_dropout = nn.Dropout(config.dropout)
        self.resid_dropout = nn.Dropout(config.dropout)
        self.dropout = config.dropout
        self.flash = hasattr(torch.nn.functional, 'scaled_dot_product_attention') and config.flash_attn # Attention 算法，通过减少 HBM（显存）的读写次数，可以大幅提升计算速度并降低显存占用从 N*N降低到N

    def forward(self, x, position_embeddings, past_key_value=None, use_cache=False, attention_mask=None):
        bsz, seq_len, _ = x.shape #B，T，dim
        xq, xk, xv = self.q_proj(x), self.k_proj(x), self.v_proj(x) #得到qkv
        xq = xq.view(bsz, seq_len, self.n_local_heads, self.head_dim) #reshape为多头
        xk = xk.view(bsz, seq_len, self.n_local_kv_heads, self.head_dim)
        xv = xv.view(bsz, seq_len, self.n_local_kv_heads, self.head_dim)
        xq, xk = self.q_norm(xq), self.k_norm(xk) #norm一下
        cos, sin = position_embeddings
        xq, xk = apply_rotary_pos_emb(xq, xk, cos, sin) #位置编码，后面再学
        if past_key_value is not None:
            xk = torch.cat([past_key_value[0], xk], dim=1) #应该是元组保存的，[0]代表k，[1]代表v
            xv = torch.cat([past_key_value[1], xv], dim=1)
        past_kv = (xk, xv) if use_cache else None#新的kv缓存
        xq, xk, xv = (xq.transpose(1, 2), repeat_kv(xk, self.n_rep).transpose(1, 2),
                      repeat_kv(xv, self.n_rep).transpose(1, 2)) #GQA，repeta一下 kv到q的size
        if self.flash and (seq_len > 1) and (not self.is_causal or past_key_value is None) and (#Flash attn
                attention_mask is None or torch.all(attention_mask == 1)):
            output = F.scaled_dot_product_attention(xq, xk, xv, dropout_p=self.dropout if self.training else 0.0,
                                                    is_causal=self.is_causal)
        else:
            scores = (xq @ xk.transpose(-2, -1)) / math.sqrt(self.head_dim) #计算缩放注意力的 scores弄了后为T，T的维度，= seq，seq
            #取出当前正在计算的序列片段的得分。结合 KV Cache 机制，scores 的 KV 维度包含了历史长度+当前长度，这里只对当前输入的 seq_len 长度部分加掩码。
            if self.is_causal: scores[:, :, :, -seq_len:] += torch.full((seq_len, seq_len), float("-inf"),
                                                                        device=scores.device).triu(1) #应用因果掩码，full一个下三角矩阵，
            if attention_mask is not None: scores += (1.0 - attention_mask.unsqueeze(1).unsqueeze(2)) * -1e9 #0替换成-1e9
            output = self.attn_dropout(F.softmax(scores.float(), dim=-1).type_as(xq)) @ xv # Softmax 归一化、Dropout 与输出
        output = output.transpose(1, 2).reshape(bsz, seq_len, -1) #reshape一下
        output = self.resid_dropout(self.o_proj(output)) #output线性变换为input的shape
        return output, past_kv
```

  input : (2, 16, 768) #x
  output: (2, 16, 384) #K
  weight: (384, 768) #Wk
model.layers.0.self_attn.v_proj
  input : (2, 16, 768) #x
  output: (2, 16, 384) #V
  weight: (384, 768) #Wv
model.layers.0.self_attn.o_proj
  input : (2, 16, 768) #o
  output: (2, 16, 768) #input.shape
  weight: (768, 768) #Wo

q_proj output 最后一维 = num_attention_heads × head_dim
k_proj output 最后一维 = num_key_value_heads × head_dim
v_proj output 最后一维 = num_key_value_heads × head_dim
o_proj output 最后一维 = hidden_size

这里的num_attention_heads × head_dim保持的与hidden_size一致，因为 head_dim =self.hidden_size // self.num_attention_heads
```



## 5. Attention 公式

$Attention(Q,K,V) = softmax(QK^T / sqrt(d)) V$



```
QK^T：计算 token 之间的相关性
sqrt(d)：缩放，避免点积过大
softmax：转成注意力权重
乘 V：按权重聚合上下文信息
```



## 6. GQA 理解

| 类型 | Q heads | K/V heads   | 特点                      |
| ---- | ------- | ----------- | ------------------------- |
| MHA  | 多个    | 和 Q 一样多 | 表达力强，但 KV cache 大  |
| GQA  | 多个    | 少于 Q      | 多个 Q 共享一组 K/V       |
| MQA  | 多个    | 1 组        | KV cache 最小，但压缩更强 |

MHA:
Q heads = K/V heads

GQA:
Q heads > K/V heads

MQA:
K/V heads = 1

MiniMind 当前配置：
MHA / GQA / MQA

KV cache 节省比例：

```
标准 MHA 每层每个 token 要保存：

K: n_heads × head_dim
V: n_heads × head_dim

GQA 每层每个 token 保存：

K: n_kv_heads × head_dim
V: n_kv_heads × head_dim


所以 KV cache 理论比例大约是：

GQA cache / MHA cache = n_kv_heads / n_heads

代码中给的是4/8大约是1/2
```

## 7. Causal Mask

作用：
防止当前位置看到未来 token。

mask 形状：
[T,T]

第 i 行表示：
第 i 个 token 能看到哪些历史 token。

所以 causal mask 的矩阵大致是：

```
允许关注的位置 = 下三角矩阵

token0: 看 token0
token1: 看 token0 token1
token2: 看 token0 token1 token2
token3: 看 token0 token1 token2 token3

写成矩阵为

1 0 0 0
1 1 0 0
1 1 1 0
1 1 1 1
```

## 8. KV Cache

作用：
缓存历史 token 的 K/V，减少自回归推理中的重复计算。

公式：
KV cache bytes = B × T × L × 2 × n_kv_heads × head_dim × bytes_per_elem

## 10. 今日结论

```
Q/K 决定注意力权重，V 提供被聚合的信息。

GQA 减少的是 K/V head 数，不是 Q head 数。

Causal mask 防止模型在训练和生成时看到未来 token。

KV cache 缓存历史 K/V，减少自回归推理中的重复计算。


x: [B,T,H]

Q: [B,n_heads,T,head_dim]
K: [B,n_kv_heads,T,head_dim]
V: [B,n_kv_heads,T,head_dim]

GQA repeat 后：
K/V: [B,n_heads,T,head_dim]

Attention 输出：
[B,n_heads,T,head_dim]

合并 + o_proj：
[B,T,H]
```
