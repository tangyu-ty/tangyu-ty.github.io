重点：register_forward_hook

`register_forward_hook` 是 PyTorch 框架中一个非常核心且实用的工具。简单来说，它是**挂载在神经网络层上的一个“监听器”或“回调函数”**，允许你在不修改模型原始代码的情况下，获取该层在前向传播时的**输出**和**输入**。

```python
import torch
from model.model_minimind import MiniMindConfig, MiniMindForCausalLM

config = MiniMindConfig()
model = MiniMindForCausalLM(config)
model.eval()

def print_shape(name):
    def hook(module, inputs, output):
        if isinstance(output, torch.Tensor):#输出是张量
            print(f"{name}: {tuple(output.shape)}")
        elif isinstance(output, tuple):#输出是元组
            print(f"{name}: tuple")
            for i, x in enumerate(output):
                if isinstance(x, torch.Tensor):
                    print(f"  output[{i}]: {tuple(x.shape)}")
        else:
            print(f"{name}: {type(output)}")#否则打印名字和类型
    return hook

model.model.embed_tokens.register_forward_hook(print_shape("embed_tokens"))#给embed加入钩子函数，钩子钩的是前面的model.model.embed_tokens.
#def print_shape(name):的name只是print一下
model.lm_head.register_forward_hook(print_shape("lm_head"))给lm_head加入钩子函数，打印lm_head

input_ids = torch.randint(0, config.vocab_size, (2, 16))

with torch.no_grad():
    outputs = model(input_ids) #模型打印两个钩子函数
```

PyTorch 规定，注册到模块上的前向钩子必须符合这个签名：

- `module`：当前挂载钩子的那一层网络（比如 `embed_tokens` 层）。
- `inputs`：这一层接收到的输入（是一个元组）。
- `output`：这一层计算后输出的结果

这是 Python 的**闭包**技巧。前面说了，PyTorch 规定 `hook` 的参数只能是 `(module, inputs, output)`。那我们想打印这是哪一层的名字（比如 “embed_tokens”），这个 `name` 变量怎么传进去呢？
通过外层函数 `print_shape(name)`，我们把 `name` 像“夹带私货”一样，绑定到了 `hook` 函数的运行环境中。这样当 `hook` 执行时，它就能访问到外层的 `name` 变量了。

## 2. 源码位置

RMSNorm:
model/model_minimind.py

```
class RMSNorm(torch.nn.Module):#没什么好树洞额，对应公式敲出来
    def __init__(self, dim: int, eps: float = 1e-5):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(dim))

    def norm(self, x):
        return x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)

    def forward(self, x):
        return (self.weight * self.norm(x.float())).type_as(x)
```



MiniMindBlock:
model/model_minimind.py

```
class MiniMindBlock(nn.Module):#没什么好树洞额，大部分对应公式，基础的NN层
    def __init__(self, layer_id: int, config: MiniMindConfig):
        super().__init__()
        self.self_attn = Attention(config)
        self.input_layernorm = RMSNorm(config.hidden_size, eps=config.rms_norm_eps)
        self.post_attention_layernorm = RMSNorm(config.hidden_size, eps=config.rms_norm_eps)
        self.mlp = FeedForward(config) if not config.use_moe else MOEFeedForward(config)

    def forward(self, hidden_states, position_embeddings, past_key_value=None, use_cache=False, attention_mask=None):
        residual = hidden_states
        hidden_states, present_key_value = self.self_attn(
            self.input_layernorm(hidden_states), position_embeddings,#过一遍attn
            past_key_value, use_cache, attention_mask
        )
        hidden_states += residual
        hidden_states = hidden_states + self.mlp(self.post_attention_layernorm(hidden_states))
        return hidden_states, present_key_value

```



final norm:

```
self.post_attention_layernorm = RMSNorm(config.hidden_size, eps=config.rms_norm_eps)
```



MiniMindModel 中的 self.norm

```
self.norm = RMSNorm(config.hidden_size, eps=config.rms_norm_eps)
```



## 3. RMSNorm 公式

RMSNorm 的核心是：

```
对每个 token 的 hidden 向量，在 hidden_size 维度上计算均方根，
然后把向量缩放到稳定范围，再乘以可学习参数 weight。
```

公式如下

$RMS(x) = sqrt(mean(x_i^2) + eps)$

$RMSNorm(x) = x / RMS(x) * weight$

注意它和 LayerNorm 不一样：

```
LayerNorm:
先减均值，再除标准差

RMSNorm:
不减均值，只除 RMSs

RMSNorm 只做缩放，不做中心化。
```

RMSNorm 原论文的核心假设就是：LayerNorm 中的 re-centering invariance，也就是减均值带来的不变性，并非总是必要；只保留 re-scaling invariance 也能稳定训练，并减少计算开销。
PyTorch 官方文档也将 `RMSNorm` 描述为对 mini-batch 输入应用 Root Mean Square Layer Normalization。

## 4. RMSNorm 参数量

RMSNorm 只有一个可学习参数：

```
self.weight = nn.Parameter(torch.ones(dim))
```

hidden_size =768
单个 RMSNorm 参数量 =768
RMSNorm 个数 = layers+final norm
RMSNorm 总参数量 =(8+1)*768 = 12,288 + 768 = 13,056

```
RMSNorm 参数量很小，不是模型参数量主要来源。
它的价值主要是稳定训练和控制激活数值范
```



## 5. Pre-Norm Block 结构

```
h = x + Attention(RMSNorm(x))
y = h + MLP(RMSNorm(h))

Attention 前先 Norm
MLP 前也先 Norm

def block_forward(x):
    h = x + self_attn(input_layernorm(x))
    y = h + mlp(post_attention_layernorm(h))
    return y
```

整体结构图如：

```
输入 x
│
├── norm_x = RMSNorm(x)
│
├── attn_out = Attention(norm_x)
│
├── h = x + attn_out
│
├── norm_h = RMSNorm(h)
│
├── mlp_out = MLP(norm_h)
│
└── y = h + mlp_out

Norm 不是放在 residual add 后面，
而是放在 Attention / MLP 前面。

我这样理解的，norm后神经网络训练就没毛刺，更平滑
```

**Pre-Norm 和 Post-Norm 对比**

```
Post-Norm:
x → Attention → Add → Norm

Pre-Norm:
x → Norm → Attention → Add
```

对比表：

| 结构      | 顺序              | 常见特点                    |
| --------- | ----------------- | --------------------------- |
| Post-Norm | 子层计算后再 Norm | 早期 Transformer 常见       |
| Pre-Norm  | 子层计算前先 Norm | 现代 LLM 更常见，训练更稳定 |

可以这样理解：

```
Pre-Norm 让 residual 主干路径更“干净”，
梯度可以更容易沿着 x → x + f(norm(x)) 这条路径传播。
```

Day 3不需要证明它，只需要能解释为什么现代 LLM 更偏好 Pre-Norm：

```
1. 深层模型更稳定
2. 梯度传播路径更直接
3. 更适合大规模自回归语言模型训练
```

## 7. RMSNorm vs LayerNorm

LayerNorm:
减均值 + 除标准差

RMSNorm:
不减均值，只除 RMS

公式对比

$RMS(x) = sqrt(mean(x_i^2) + eps)$

$RMSNorm(x) = x / RMS(x) * weight$

vs

$\mu = mean(x_i)$
$\sigma = sqrt(mean((x_i - \mu)^2) + eps)$
$LayerNorm(x) = (x - \mu) / \sigma * weight + bias$

```
def repeat_kv(x: torch.Tensor, n_rep: int) -> torch.Tensor: #repeat一下，没什么复杂的
    bs, slen, num_key_value_heads, head_dim = x.shape
    if n_rep == 1: return x
    return (x[:, :, :, None, :].expand(bs, slen, num_key_value_heads, n_rep, head_dim).reshape(bs, slen,
                                                                                               num_key_value_heads * n_rep,
                                                                                               head_dim))

class Attention(nn.Module):
    def __init__(self, config: MiniMindConfig):
        super().__init__()
        # 其中 num_key_value_heads 如果为空会等于 num_attention_heads，否则就形成 GQA 结构
        # 如果配置中没有指定（None），则kv默认与 Q 的头数量（num_attention_heads）相同，当它小于 Q 的头数时，就是 GQA；当它等于 1 时，就是 MQA。
        self.num_key_value_heads = config.num_attention_heads if config.num_key_value_heads is None else config.num_key_value_heads
        self.n_local_heads = config.num_attention_heads  # Q的头的数量
        self.n_local_kv_heads = self.num_key_value_heads  # KV 的头数量
        self.n_rep = self.n_local_heads // self.n_local_kv_heads  # 因为 Q 的头数通常多于 KV 的头数，所以在计算 Attention 时，需要将 KV 的特征在头维度上复制，以和 Q 的头数对齐。
        self.head_dim = config.head_dim  # 每个注意力头的维度大小 #总隐藏维度 = num_attention_heads * head_dim。
        self.is_causal = True #设为 True 表示在解码时，当前 Token 只能看到前面的 Token，不能看到未来的 Token
        self.q_proj = nn.Linear(config.hidden_size, config.num_attention_heads * self.head_dim, bias=False)  # 线性映射层得到q
        self.k_proj = nn.Linear(config.hidden_size, self.num_key_value_heads * self.head_dim, bias=False)
        self.v_proj = nn.Linear(config.hidden_size, self.num_key_value_heads * self.head_dim, bias=False)
        self.o_proj = nn.Linear(config.num_attention_heads * self.head_dim, config.hidden_size, bias=False) #前面的都是单个head里面的qkv，这个就是把多个head拼接起来并返回hidden_size
        self.q_norm = RMSNorm(self.head_dim, eps=config.rms_norm_eps)
        self.k_norm = RMSNorm(self.head_dim, eps=config.rms_norm_eps)
        self.attn_dropout = nn.Dropout(config.dropout)
        self.resid_dropout = nn.Dropout(config.dropout)
        self.dropout = config.dropout
        self.flash = hasattr(torch.nn.functional, 'scaled_dot_product_attention') and config.flash_attn # Attention 算法，通过减少 HBM（显存）的读写次数，可以大幅提升计算速度并降低显存占用从 N*N降低到N

    def forward(self, x, position_embeddings, past_key_value=None, use_cache=False, attention_mask=None):
        bsz, seq_len, _ = x.shape #B，T，dim
        xq, xk, xv = self.q_proj(x), self.k_proj(x), self.v_proj(x) #得到qkv
        xq = xq.view(bsz, seq_len, self.n_local_heads, self.head_dim) #reshape为多头
        xk = xk.view(bsz, seq_len, self.n_local_kv_heads, self.head_dim)
        xv = xv.view(bsz, seq_len, self.n_local_kv_heads, self.head_dim)
        xq, xk = self.q_norm(xq), self.k_norm(xk) #norm一下
        cos, sin = position_embeddings
        xq, xk = apply_rotary_pos_emb(xq, xk, cos, sin) #位置编码，后面再学
        if past_key_value is not None:
            xk = torch.cat([past_key_value[0], xk], dim=1) #应该是元组保存的，[0]代表k，[1]代表v
            xv = torch.cat([past_key_value[1], xv], dim=1)
        past_kv = (xk, xv) if use_cache else None#新的kv缓存
        xq, xk, xv = (xq.transpose(1, 2), repeat_kv(xk, self.n_rep).transpose(1, 2),
                      repeat_kv(xv, self.n_rep).transpose(1, 2)) #GQA，repeta一下 kv到q的size
        if self.flash and (seq_len > 1) and (not self.is_causal or past_key_value is None) and (#Flash attn
                attention_mask is None or torch.all(attention_mask == 1)):
            output = F.scaled_dot_product_attention(xq, xk, xv, dropout_p=self.dropout if self.training else 0.0,
                                                    is_causal=self.is_causal)
        else:
            scores = (xq @ xk.transpose(-2, -1)) / math.sqrt(self.head_dim) #计算缩放注意力的 scores弄了后为T，T的维度，= seq，seq
            #取出当前正在计算的序列片段的得分。结合 KV Cache 机制，scores 的 KV 维度包含了历史长度+当前长度，这里只对当前输入的 seq_len 长度部分加掩码。
            if self.is_causal: scores[:, :, :, -seq_len:] += torch.full((seq_len, seq_len), float("-inf"),
                                                                        device=scores.device).triu(1) #应用因果掩码，full一个下三角矩阵，
            if attention_mask is not None: scores += (1.0 - attention_mask.unsqueeze(1).unsqueeze(2)) * -1e9 #0替换成-1e9
            output = self.attn_dropout(F.softmax(scores.float(), dim=-1).type_as(xq)) @ xv # Softmax 归一化、Dropout 与输出
        output = output.transpose(1, 2).reshape(bsz, seq_len, -1) #reshape一下
        output = self.resid_dropout(self.o_proj(output)) #output线性变换为input的shape
        return output, past_kv
```

## 5. Attention 公式

$Attention(Q,K,V) = softmax(QK^T / sqrt(d)) V$



```
QK^T：计算 token 之间的相关性
sqrt(d)：缩放，避免点积过大
softmax：转成注意力权重
乘 V：按权重聚合上下文信息
```



## 6. GQA 理解

| 类型 | Q heads | K/V heads   | 特点                      |
| ---- | ------- | ----------- | ------------------------- |
| MHA  | 多个    | 和 Q 一样多 | 表达力强，但 KV cache 大  |
| GQA  | 多个    | 少于 Q      | 多个 Q 共享一组 K/V       |
| MQA  | 多个    | 1 组        | KV cache 最小，但压缩更强 |

MHA:
Q heads = K/V heads

GQA:
Q heads > K/V heads

MQA:
K/V heads = 1

MiniMind 当前配置：
MHA / GQA / MQA

KV cache 节省比例：

```
标准 MHA 每层每个 token 要保存：

K: n_heads × head_dim
V: n_heads × head_dim

GQA 每层每个 token 保存：

K: n_kv_heads × head_dim
V: n_kv_heads × head_dim


所以 KV cache 理论比例大约是：

GQA cache / MHA cache = n_kv_heads / n_heads

代码中给的是4/8大约是1/2
```

```python

#这个方法的目的：提前算好每个位置、每个维度需要旋转的角度，并转换为 cos 和 sin 值。
def precompute_freqs_cis(dim: int, end: int = int(32 * 1024), rope_base: float = 1e6, rope_scaling: dict = None):
    #不同维度旋转的频率不同。低维度转得快（高频），高维度转得慢（低频）。
    #这里算出来的 freqs 是一个长度为 dim // 2 的一维张量，就是每个维度对的基础角速度。
    freqs, attn_factor = 1.0 / (rope_base ** (torch.arange(0, dim, 2)[: (dim // 2)].float() / dim)), 1.0

    if rope_scaling is not None:  # YaRN: f'(i) = f(i)((1-γ) + γ/s), where γ∈[0,1] is linear ramp  #处理 YaRN 长文本外推
        orig_max, factor, beta_fast, beta_slow, attn_factor = (
            rope_scaling.get("original_max_position_embeddings", 2048), rope_scaling.get("factor", 16),
            rope_scaling.get("beta_fast", 32.0), rope_scaling.get("beta_slow", 1.0),
            rope_scaling.get("attention_factor", 1.0)
        )
        if end / orig_max > 1.0:
            inv_dim = lambda b: (dim * math.log(orig_max / (b * 2 * math.pi))) / (2 * math.log(rope_base))
            low, high = max(math.floor(inv_dim(beta_fast)), 0), min(math.ceil(inv_dim(beta_slow)), dim // 2 - 1)
            ramp = torch.clamp((torch.arange(dim // 2, device=freqs.device).float() - low) / max(high - low, 0.001), 0,
                               1)
            freqs = freqs * (1 - ramp + ramp / factor)
    t = torch.arange(end, device=freqs.device) #生成 position 序列,由max_seq决定
    freqs = torch.outer(t, freqs).float() #外积生成矩阵，位置x频率
    #拼接的原因是上面球freqs的时候torch.arange(0, dim, 2),所以这里需要拼接，让维度从dim/2->dim	
    freqs_cos = torch.cat([torch.cos(freqs), torch.cos(freqs)], dim=-1) * attn_factor
    freqs_sin = torch.cat([torch.sin(freqs), torch.sin(freqs)], dim=-1) * attn_factor
    return freqs_cos, freqs_sin
```

```python

def apply_rotary_pos_emb(q, k, cos, sin, unsqueeze_dim=1):
    # print("q before rope:", q.shape)
    # print("k before rope:", k.shape)
    # print("freqs_cos:", cos.shape)
    # print("freqs_sin:", sin.shape)
    
    def rotate_half(x): return torch.cat((-x[..., x.shape[-1] // 2:], x[..., : x.shape[-1] // 2]), dim=-1)

    q_embed = ((q * cos.unsqueeze(unsqueeze_dim)) + (rotate_half(q) * sin.unsqueeze(unsqueeze_dim))).to(q.dtype)
    k_embed = ((k * cos.unsqueeze(unsqueeze_dim)) + (rotate_half(k) * sin.unsqueeze(unsqueeze_dim))).to(k.dtype)

    # print("q after rope:", q_embed.shape)
    # print("k after rope:", k_embed.shape)
    return q_embed, k_embed
```

这里重点理解rotate_half：

```
假设：

x = [a, b, c, d]

那么：

x 前半部分 = [a, b]
x 后半部分 = [c, d]

rotate_half(x) 得到：

[-c, -d, a, b]

所以：

rotate_half([a,b,c,d]) = [-c,-d,a,b]

这对应一种“半维分组”的旋转写法。它不是把 [a,b]、[c,d] 分别旋转，而是把前半和后半配对：

(a, c)
(b, d)

如果：

x = [a, b, c, d]
cos = [cos0, cos1, cos0, cos1]
sin = [sin0, sin1, sin0, sin1]

那么：

x * cos + rotate_half(x) * sin

等于：

[
  a*cos0 + (-c)*sin0,
  b*cos1 + (-d)*sin1,
  c*cos0 + a*sin0,
  d*cos1 + b*sin1
]

也就是：

[
  a*cos0 - c*sin0,
  b*cos1 - d*sin1,
  c*cos0 + a*sin0,
  d*cos1 + b*sin1
]

这正是二维旋转：

(a, c) 旋转 θ0
(b, d) 旋转 θ1

所以你可以记：

rotate_half 不是随便换位置；
它是在构造二维旋转公式里的 [-x2, x1]。
```

### 标准二维旋转

二维旋转的公式主要有两种等价的表示形式，RoPE 用的就是后者：

在二维平面上，将点 $(x, y)$ 绕原点逆时针旋转角度 $\theta$，得到新点 $(x', y')$ 的公式为：
$$
\begin{pmatrix} x' \\ y' \end{pmatrix} =
\begin{pmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{pmatrix}
\begin{pmatrix} x \\ y \end{pmatrix}
$$
展开后的代数形式（RoPE 直接使用的形式）

将上面矩阵乘法展开：
$$ x' = x \cos\theta - y \sin\theta $$
$$ y' = x \sin\theta + y \cos\theta $$

## 4. 为什么作用于 Q/K

$Attention(Q,K,V) = softmax(QK^T / sqrt(d)) V$

```
RoPE 作用于 Q/K，是因为 attention score 由 QK^T 决定。
如果 Q/K 带有位置信息，那么 attention 权重自然具有相对位置感知能力。
V 不参与注意力权重计算，所以通常不加 RoPE。
```

## 6. start_pos 理解

```
prefill:
start_pos = 0

decode:
start_pos = cached_len
```

作用：
保证新 token 使用全文中的真实位置。

start_pos 是 RoPE 与 KV cache 增量推理之间的桥梁。
没有 start_pos，新生成 token 的位置会被错误地当成 0。

```
past_key_values 为空：
start_pos = 0
freqs = freqs[0:T]

past_key_values 不为空：
start_pos = cached_len
freqs = freqs[cached_len : cached_len + T_new]
```



## 7. rope_theta

默认值 rope_theta=1e-6

含义：
RoPE 频率基底。

影响：
控制不同维度的旋转频率分布。

rope_theta 越大，低频成分变化更慢；
它会影响模型对长距离位置关系的编码方式。

## 8. rope_scaling / YaRN

```
rope_scaling 的目的：
让模型在超过原始训练长度时，尽量保持 RoPE 位置模式可用。

RoPE 虽然比传统绝对位置 embedding 更适合长度外推，但模型如果训练时只见过较短上下文，直接推理到更长上下文仍可能退化。

YaRN 论文指出，使用 RoPE 的模型在超过训练时上下文长度后会难以泛化；YaRN 是一种高效扩展上下文窗口的方法，论文称相比此前方法可用更少 token 和训练步数完成上下文扩展。
```

| 项目                 | 原始 RoPE    | RoPE Scaling / YaRN    |
| -------------------- | ------------ | ---------------------- |
| 目标                 | 编码位置信息 | 扩展上下文长度         |
| 是否改变模型参数     | 否           | 通常不直接改变模型权重 |
| 是否影响 cos/sin     | 是           | 是                     |
| 是否影响推理最大长度 | 间接影响     | 明显影响               |
| 是否一定无损         | 否           | 否，需要评估           |

```
class FeedForward(nn.Module): #简单的门控神经网络，没啥说的，GLU思想，维度变化hidden_size, intermediate, size.hidden_size
    def __init__(self, config: MiniMindConfig, intermediate_size: int = None):
        super().__init__()
        intermediate_size = intermediate_size or config.intermediate_size
        self.gate_proj = nn.Linear(config.hidden_size, intermediate_size, bias=False)
        self.down_proj = nn.Linear(intermediate_size, config.hidden_size, bias=False)
        self.up_proj = nn.Linear(config.hidden_size, intermediate_size, bias=False)#门控，计算
        self.act_fn = ACT2FN[config.hidden_act]

    def forward(self, x):
        return self.down_proj(self.act_fn(self.gate_proj(x)) * self.up_proj(x))
```

```python
class MOEFeedForward(nn.Module):
    def __init__(self, config: MiniMindConfig):
        super().__init__()
        self.config = config
        self.gate = nn.Linear(config.hidden_size, config.num_experts, bias=False) #门控线性层
        self.experts = nn.ModuleList(
            [FeedForward(config, intermediate_size=config.moe_intermediate_size) for _ in range(config.num_experts)])#由FFN组成的专家
        self.act_fn = ACT2FN[config.hidden_act] #默认silu，从transform包中拿到

    def forward(self, x):
        batch_size, seq_len, hidden_dim = x.shape
        x_flat = x.view(-1, hidden_dim) #每个token的维度
        scores = F.softmax(self.gate(x_flat), dim=-1)#每个token的专家得分
        topk_weight, topk_idx = torch.topk(scores, k=self.config.num_experts_per_tok, dim=-1, sorted=False)#取出每个token的前k个专家得分
        if self.config.norm_topk_prob: topk_weight = topk_weight / (topk_weight.sum(dim=-1, keepdim=True) + 1e-20)# 归一化Top-K权重，使其和为1，确保输出量级不变
        y = torch.zeros_like(x_flat)
        for i, expert in enumerate(self.experts):#遍历专家
            mask = (topk_idx == i) #找出当前专家i被哪些token的Top-K选中，这是一个[token]的向量
            if mask.any():#如果由专家被选中
                token_idx = mask.any(dim=-1).nonzero().flatten() # 获取选中当前专家的token的原始索引
                weight = topk_weight[mask].view(-1, 1)# 提取这些token对应当前专家的权重
                # 取出这个专家要计算的token_idx，将专家输出乘以权重，累加到对应token的位置，index_add_累加
                y.index_add_(0, token_idx, (expert(x_flat[token_idx]) * weight).to(y.dtype))
            elif self.training:
                # 【保活技巧】训练时，若某专家没被任何token选中，执行0*参数的操作
                # 目的是让该专家参数参与计算图，防止分布式训练(DDP)因无梯度而报错
                # 这个 y[0, 0]是第一个token的第一个维度的值，也可以是任何值，随便挑的
                y[0, 0] += 0 * sum(p.sum() for p in expert.parameters())
        # 【负载均衡】计算辅助损失，防止所有token都只选少数几个专家(路由崩塌)
        if self.training and self.config.router_aux_loss_coef > 0:
            # 每个专家被选中的频率，
            # 假设有两个token，第一个token选择2个专家[0，2]，第二个token选择了两个专家[2,3]
            # load的第一个值，就是[[1,0,0,0],[0,0,1,0]]，代表第一个token的onehot数据量
            # 然后mean一下球的频率
            load = F.one_hot(topk_idx, self.config.num_experts).float().mean(0)
            self.aux_loss = (load * scores.mean(0)).sum() * self.config.num_experts * self.config.router_aux_loss_coef # 频率*得分，越大惩罚越重
        else:
            self.aux_loss = scores.new_zeros(1).squeeze()
        return y.view(batch_size, seq_len, hidden_dim)# 恢复原始形状
```

## 3. Dense FeedForward 结构

```

gate_proj:
[B,T,H] → [B,T,I]

up_proj:
[B,T,H] → [B,T,I]

逐元素相乘：
[B,T,I]

down_proj:
[B,T,I] → [B,T,H]

输出：
[B,T,H]
```

这个没什么说的，普通的GLU的神经网络。

## 4. SwiGLU 公式

最早的FFN是$FFN(x) = W_{down}(activation(W_{up} x))$

$FFN(x) = W_down( SiLU(xW_gate) ⊙ xW_up )$

$⊙$ 表示逐元素相乘 $SiLU(x) = x · sigmoid(x)$

如果 hidden_act 不是 SiLU，则把 SiLU 替换为源码配置中的 act_fn。

## 6. Attention vs FFN

假设不考虑 bias。

单层 Attention 参数量 =

```
q_proj: H × (n_heads × head_dim)
k_proj: H × (n_kv_heads × head_dim)
v_proj: H × (n_kv_heads × head_dim)
o_proj: (n_heads × head_dim) × H

如果
n_heads × head_dim = H

那么：

q_proj ≈ H²
o_proj ≈ H²

如果是 MHA：

k_proj ≈ H²
v_proj ≈ H²

Attention params ≈ 4H²

如果是 GQA：

k_proj + v_proj 更小
Attention params < 4H²

比如：

H = 768
n_heads = 8
head_dim = 96
n_kv_heads = 2

那么：

q_proj = 768 × 768 = 589,824
k_proj = 768 × 192 = 147,456
v_proj = 768 × 192 = 147,456
o_proj = 768 × 768 = 589,824

Attention params = 1,474,560 ≈ 1.47M / layer
```

单层 FFN 参数量 =

```
FFN params = 3 × head_dim(768) × 2048
           = 4,718,592
           ≈ 4.72M / layer
```



FFN / Attention =4.72/1.47

结论：FFN 参数量约为 Attention 的 3.2 倍

一些关于 LLM 剪枝/稀疏的研究也指出，在使用 MQA/GQA 的现代 LLM 中，SwiGLU-based MLP 可能占据非常高的参数比例，因此 MLP 压缩是模型压缩的重要对象

## 8. MoE 理解

Dense FFN:
每个 token 经过同一个 FFN。

MoE FFN:
每个 token 先经过 gate/router；
router 选择 top-k 个 expert；
token 只进入被选中的专家；
最后按 gate 权重合并专家输出。

```
x: [B,T,H]
↓
router / gate
↓
top-k experts
↓
expert_i(x)
↓
weighted sum
↓
output: [B,T,H]
```

MoE 的核心不是让每个 token 用所有参数，
而是让不同 token 动态选择不同子网络。

## 10. 今日结论

FeedForward 是逐 token 的非线性变换，不负责 token 之间的信息交互。

SwiGLU 使用 gate_proj 和 up_proj 两条分支，逐元素相乘后再 down_proj 回 hidden_size。

FFN 参数量约为 3 × hidden_size × intermediate_size，通常是 Transformer Block 的参数大户。

MoE 把一个大 FFN 拆成多个 experts，每个 token 只激活 top-k 个 experts。

Attention:
负责 token 与 token 之间的信息交互

FeedForward:
负责每个 token 内部表示的非线性变换

## 8. 结构改动实验

```
实验 A：改变 intermediate_size
```

## 9. 最终理解

1. Embedding 负责 token id 到 hidden vector。
2. Attention 负责 token 间信息交互。
3. RoPE 负责给 Q/K 注入位置信息。
4. GQA 通过减少 K/V heads 降低 KV cache。
5. FeedForward 负责逐 token 非线性变换。
6. LM Head 负责 hidden vector 到 vocab logits。

```
MiniMind = Decoder-only Transformer

入口：
token id → embedding

主体：
多层 Pre-Norm Transformer Block

每个 Block：
Attention 负责 token 间交互
FeedForward 负责 token 内非线性变换

位置：
RoPE 作用在 Q/K

效率：
GQA 降低 K/V heads 和 KV cache

出口：
hidden states → lm_head → vocab logits
```
