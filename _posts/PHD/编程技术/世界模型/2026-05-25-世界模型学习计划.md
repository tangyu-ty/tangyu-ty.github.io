# 主线任务

世界模型中的三条主线

| 能力    | 世界模型中的含义               | 与语义通信的对应关系   |
| ----- | ---------------------- | ------------ |
| 表征学习  | 将图像/视频压缩为 latent state | 发送端提取语义表示    |
| 动态预测  | 根据当前状态与动作预测未来状态        | 接收端恢复或预测未来语义 |
| 规划/决策 | 在 latent space 中寻找最优动作 | 衡量通信是否真正支持任务 |

V-JEPA2包含大规模视频自监督模型与 action-conditioned 机器人世界模型；官方仓库提供的模型规模从 80M 到 2B 参数，更适合作为后期研究参考


所以我的学习路线是

```text
经典 World Models
      ↓
理解 latent dynamics 与 imagined rollout
      ↓
DINO-WM：在预训练视觉 latent 上预测未来
      ↓
JEPA / V-JEPA：预测表示而不是重构像素
      ↓
加入信道层，形成预测式语义通信系统
```

### 你读这篇论文时需要回答的 6 个问题



```
1.为什么世界模型不直接预测动作，而是先学习环境动力学？

是因为其核心目标是学习一个压缩的时空表示来模拟环境，从而让控制器能够基于该模型学习策略[2]。 如果直接预测动作，模型将无法生成反事实的“梦境”来训练智能体；而通过学习动力学，智能体可以完全在自己的世界模型生成的幻觉梦境中进行训练，并将学到的策略迁移回真实环境中[2]。 此外，学习动力学模型使得智能体能够像人类一样，基于对未来的预测做出快速的本能反应，而不需要有意识地推演未来的各种场景[3],[5]。



2. 为什么要用 VAE 表示图像，而不是直接用 CNN 特征？

使用 VAE 表示图像而不是直接用 CNN 特征，是因为VAE能够将高维的视觉输入压缩成一个小型的抽象潜在向量z，这种有损压缩过程能够捕捉图像帧的本质特征[13],[15],[30]。 直接使用 CNN 特征可能缺乏这种高度压缩和抽象的瓶颈结构，而 VAE 的编码-解码机制迫使模型学习到最具代表性的潜在空间[61]。 不过文档也指出，使用 VAE 作为独立的 V 模型存在局限性，它可能会编码与任务无关的观察细节（例如在赛车环境中忽略了道路上的关键瓷砖，却还原了不重要的砖墙纹理），而无监督学习本质上无法预知哪些特征对任务有用[64]。

3. MDN-RNN 输出的为什么是概率分布，而不是确定性下一帧？

MDN-RNN 输出概率分布而不是确定性的下一帧，是因为许多复杂环境本质上是随机的，确定性的预测无法捕捉环境的不确定性[16]。 更重要的是，输出概率分布（高斯混合分布）可以有效防止控制器利用世界模型的缺陷[54]。 如果使用确定性模型，智能体很容易找到在模型中表现极佳但在真实环境中失效的“对抗性策略”（例如在 VizDoom 中通过移动让怪物永远不发射火球）[52],[54]。 概率分布中的离散模式还能更好地建模具有随机离散事件的环境逻辑（如怪物是否发射火球），同时允许通过调节温度参数 
τ 来控制模型的随机性，从而在真实感和可利用性之间进行权衡[54]。


4. `z_t` 与 RNN hidden state `h_t` 分别表示什么？

在模型架构中，zt表示视觉模型（V）在当前时间步 
t对观察图像压缩后的潜在空间向量，它仅捕捉当前时刻的观察表示，缺乏预测能力[15],[35]。 而 RNN 的隐藏状态 ht是记忆模型（M）的状态，它基于历史信息生成，包含了关于未来 z 向量概率分布的信息，因此代表了智能体对未来的期望和预测[16],[35]。 将 zt和ht结合，控制器就能同时获得当前的观察表示和对未来的预测表示[35]。

5. Controller 为什么可以非常小？

Controller（C）可以非常小，是因为大部分的模型复杂度和参数都被转移到了世界模型（V 和 M）中[9],[21]。 V 和 M 模型可以通过反向传播算法在现代 GPU 上高效训练，而将复杂性留给世界模型后，C 只需作为一个简单的单层线性模型，将 zt和 ht 映射为动作 at[21]。 这种极简设计使得搜索空间非常小，从而让进化策略（如 CMA-ES）等非传统优化方法能够有效解决强化学习中的信用分配问题[21],[24]。

6. 如果把真实环境替换为无线通信系统，哪些变量可以映射为 semantic latent、channel noise 和 receiver state？


如果将真实环境替换为无线通信系统，根据文档中世界模型的变量语义，可以进行如下映射： 
1. Semantic Latent（语义潜在表示）：可映射为 zt。在通信系统中，它对应发送端原始信号经过信源编码后提取的核心语义特征，即去除了冗余后的低维压缩表示[13],[15]。 
2. Channel Noise（信道噪声）：可映射为 MDN-RNN 输出的概率分布中的随机性以及温度参数 τ 控制的不确定性。在通信系统中，这对应信号在传输过程中受到的信道衰落和加性噪声，导致接收端状态具有随机性而非确定性[16],[47]。 
3. Receiver State（接收端状态）：可映射为 RNN 的隐藏状态 ht。在通信系统中，它对应接收端基于历史接收信号和信道状态信息对当前信道和信号进行的估计与记忆，包含了对未来信号分布的预测信息[16],[35]。


```
